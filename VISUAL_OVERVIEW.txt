│ KEY METHODS                                                                │
└────────────────────────────────────────────────────────────────────────────┘

ORDER TRACKING SERVICE
═══════════════════════════════════════════════════════════════════════════════

  createOrder()              → Creates new order (Status: pending)
  vendorAcceptOrder()        → Vendor accepts (Status: accepted)
  vendorRejectOrder()        → Vendor rejects (Status: rejected)
  vendorMarkReady()          → Vendor marks ready (Status: ready_for_pickup)
  riderAcceptOrder()         → Rider accepts (Status: rider_accepted)
  riderPickedUpOrder()       → Rider picks up (Status: delivering)
  completeDelivery()         → Order delivered (Status: delivered)
  cancelOrder()              → Order cancelled
  
  getOrder()                 → Get order details
  getUserOrders()            → Get user's orders
  getVendorOrders()          → Get vendor's orders
  getRiderOrders()           → Get rider's orders
  
  subscribeToOrder()         → Subscribe to order updates


NOTIFICATION SERVICE
═══════════════════════════════════════════════════════════════════════════════

  notifyOrderPlaced()        → Vendor gets order notification
  notifyOrderAccepted()      → User gets accepted notification
  notifyOrderReady()         → User + Rider get ready notification
  notifyRiderAccepted()      → User + Vendor get rider assigned notification
  notifyOrderDelivered()     → User + Vendor get delivered notification
  
  sendMessage()              → Send message between parties
  getMessages()              → Get message history
  getNotifications()         → Get notifications for user
  markAsRead()               → Mark notification as read
  
  subscribe()                → Subscribe to notification events
